# Claude Mac Environment Implementation Plan — Phase 1

**Goal:** Create the base Docker image and minimal Dev Container config so Claude Code runs inside a container with proper filesystem mounts.

**Architecture:** Ubuntu-based Dockerfile with build arg for distro selection, non-root `claude` user, Node.js LTS via nodesource, Claude Code via npm. Minimal hardcoded devcontainer.json with bind mounts for project dirs (RW) and dotfiles (RO).

**Tech Stack:** Docker, Ubuntu 24.04, Node.js 24 LTS, Python 3.x, Claude Code CLI (@anthropic-ai/claude-code)

**Scope:** Phase 1 of 8 from original design

**Codebase verified:** 2026-03-29 — repo at /tmp/claude-mac-env contains LICENSE, docs/ only. No Dockerfile, .devcontainer/, or .gitignore exist yet.

### Task 2: Create the Dockerfile

**Files:**
- Create: `Dockerfile`

**Step 1: Create the Dockerfile**

```dockerfile
# Base image — Ubuntu by default, overridable via build arg
ARG BASE_IMAGE=ubuntu:24.04
FROM ${BASE_IMAGE}

# Prevent interactive prompts during package install
ENV DEBIAN_FRONTEND=noninteractive

# Core system packages
RUN apt-get update && apt-get install -y --no-install-recommends \
    git \
    curl \
    wget \
    jq \
    build-essential \
    ca-certificates \
    gnupg \
    sudo \
    python3 \
    python3-pip \
    python3-venv \
    openssh-client \
    && rm -rf /var/lib/apt/lists/*

# Install Node.js LTS via nodesource
ARG NODE_MAJOR=24
RUN curl -fsSL https://deb.nodesource.com/setup_${NODE_MAJOR}.x | bash - \
    && apt-get install -y nodejs \
    && rm -rf /var/lib/apt/lists/*

# Create non-root user with sudo access
ARG USERNAME=claude
ARG USER_UID=1000
ARG USER_GID=${USER_UID}
RUN groupadd --gid ${USER_GID} ${USERNAME} \
    && useradd --uid ${USER_UID} --gid ${USER_GID} -m -s /bin/bash ${USERNAME} \
    && echo "${USERNAME} ALL=(root) NOPASSWD:ALL" > /etc/sudoers.d/${USERNAME} \
    && chmod 0440 /etc/sudoers.d/${USERNAME}

# Install Claude Code CLI globally
RUN npm install -g @anthropic-ai/claude-code

# Create workspaces directory for project mounts
RUN mkdir -p /workspaces && chown ${USERNAME}:${USERNAME} /workspaces

# Shared utility for distro detection (used by Features later)
COPY detect-package-manager.sh /usr/local/bin/detect-package-manager.sh
RUN chmod +x /usr/local/bin/detect-package-manager.sh

# Switch to non-root user
USER ${USERNAME}
WORKDIR /home/${USERNAME}

# Verify installations
RUN node --version && npm --version && python3 --version && git --version && claude --version
```

**Step 2: Create the detect-package-manager.sh utility**

This shared utility will be used by Dev Container Features in later phases. Create it now so the Dockerfile doesn't fail.

**Files:**
- Create: `detect-package-manager.sh`

```bash
#!/usr/bin/env bash
# Detect the package manager for the current Linux distro.
# Used by Dev Container Feature install.sh scripts.
# Returns: apt, dnf, apk, or pacman. Exits 1 if unknown.

set -euo pipefail

if command -v apt-get &>/dev/null; then
    echo "apt"
elif command -v dnf &>/dev/null; then
    echo "dnf"
elif command -v apk &>/dev/null; then
    echo "apk"
elif command -v pacman &>/dev/null; then
    echo "pacman"
else
    echo "unknown" >&2
    exit 1
fi
```

**Step 3: Verify the image builds**

Run: `cd /tmp/claude-mac-env && docker build -t claude-mac-env:dev .`
Expected: Build completes successfully. Final `RUN` step prints version numbers for node, npm, python3, git, and claude.

**Step 4: Commit**

```bash
git add Dockerfile detect-package-manager.sh
git commit -m "feat: add base Dockerfile with Ubuntu, Node.js, Python, Claude Code CLI"
```

### Task 3: Create minimal devcontainer.json

**Files:**
- Create: `.devcontainer/devcontainer.json`

Note: This is a minimal hardcoded config for Phase 1 validation. Phase 6 replaces it with a template-generated version. We add it to .gitignore (already done in Task 1) because the final version will be generated per-user, but we commit this initial version to bootstrap development.

**Step 1: Create .devcontainer directory and config**

```json
{
  "name": "Claude Dev Environment",
  "build": {
    "dockerfile": "../Dockerfile",
    "context": ".."
  },
  "remoteUser": "claude",
  "mounts": [
    "source=${localEnv:HOME}/.gitconfig,target=/home/claude/.gitconfig,type=bind,readonly",
    "source=${localEnv:HOME}/.ssh,target=/home/claude/.ssh,type=bind,readonly"
  ],
  "workspaceFolder": "/workspaces",
  "workspaceMount": "source=${localWorkspaceFolder},target=/workspaces,type=bind",
  "customizations": {
    "vscode": {
      "extensions": [
        "anthropics.claude-code"
      ]
    }
  },
  "postCreateCommand": "echo 'Claude Dev Environment ready. Claude Code version:' && claude --version"
}
```

**Step 2: Verify VS Code can open the container**

Expected: VS Code opens. A notification appears: "Folder contains a Dev Container configuration file. Reopen folder to develop in a container." Click "Reopen in Container." The container builds and VS Code connects.
